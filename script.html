Script type , async , defer

  1. (Type) is the by default value (it runs HTML but when it encounter script tag pauses the HTML parsing at that time and start fetching and runing script tag same time when it done with fetching then again start parsing HTML.)
  2. (Async) Its not giving any gureented of order of execution of code so its helpful when order doesn't matter.
  3. It starts parsing HTML and side by side it also fetching data from network .. when data present in browser execution it again starts parsing HTML data...
  4. (defer) It gives fully gureented of order of execution of code
  5. Html and data fetching from network is going side by side.. but execution starts only when HTML parsing is done completly.....
  6. So, when you want order of execution use defer
  7. If you don't care about order of execution so you can use async...
